경로 하드코딩 없는 portable 파이프라인. 합성 데모로 즉시 실행되고, 실데이터는 --csv로 연결합니다.

💬 풀 QnA 앱 (Vercel) — TTT · STS · FTF

참고 앱 패턴을 그대로 포팅한 배포용 QnA. TTT=OpenAI chat, STS=OpenAI Realtime(음성↔음성, WebRTC), FTF=LiveAvatar.com(LiveKit 라이브 아바타). 키는 전부 process.env(노출 0).

Vercel 배포: qna-app/ 업로드 + 환경변수
  OPENAI_API_KEY      # TTT + STS
  LIVEAVATAR_API_KEY  # FTF (LiveAvatar.com)
  AVATAR_ID           # FTF 아바타 UUID
▲ Deploy to Vercel (원클릭) 📂 qna-app/

버튼 클릭 → GitHub 연결 → Root Directory를 qna-app 확인 → 환경변수 3개 입력 → Deploy. (LIVEAVATAR_API_KEY = 본인 HEYGEN_LIVE_API_KEY 값, AVATAR_ID는 코드에 dkpark 기본 내장)

GitHub Pages는 정적이라 서버리스 api가 안 돌아갑니다 — Vercel에 올려야 TTT/STS/FTF가 실작동. 현재 사이트의 qna.html은 정적 데모(TTT·STS 브라우저, FTF는 토큰서버 필요).
